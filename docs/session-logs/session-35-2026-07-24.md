## Key decisions

- **Ordered step list, not a new selection mechanism.** Each step is
  `{ region, generatorId, params }` — pin/lock's existing snapshot shape,
  accumulated into a list instead of held as one active selection. Reuses
  pin/lock's marquee-drag, the generator catalog, and the narrow-form
  primitive rather than inventing new authoring UI; the only new surface
  is the step list itself (add/remove/reorder).
- **Auto-connect in step order, no manual point-picking.** Grounded in a
  real fact found during exploration: 3 of the 4 primitives
  (`carveBsp`, `collapseWfc`, `partitionBiomes`) already return a
  consumable `entryPoint` (BSP also returns per-room `center`s);
  `carveCellularAutomata` returns nothing itself but its existing
  generator wrapper already derives one via `nearestOpenCell`, so the tool
  does the same inline. `connectCorridor` runs automatically between each
  consecutive step's points. User confirmed this doesn't preclude authors
  hand-writing their own composed `generatorFn` directly — the tool is a
  convenience alternative, not the only path, worth stating explicitly in
  the doc.
- **Emits a plain `.js` module, not data** — a single default-exported
  `generatorFn(zone, rng, options)` at `src/generators/composed/<name>.js`,
  matching every other generator's signature. No precedent transferred
  directly from the map editor's export (`mapEditorExport.js` emits plain
  JSON data, not logic); only the *pattern* (pure-logic module, path/name
  validation helpers, shared file-write API) carried over.
- **Overwrite allowed, but loud.** User's refinement on the third
  question: an author should be able to overwrite an existing composed
  generator, as long as it isn't silent. Resolved via the shared file-write
  API's existing `/exists` endpoint (session 29 — no new API needed): the
  tool checks before writing and surfaces an explicit confirmation if the
  target path already exists.
- **Small core-export prerequisite, done this session.** `carveBsp`,
  `carveCellularAutomata`, `collapseWfc`, `partitionBiomes`,
  `connectCorridor` were internal-only, not on `@glyphrogue/core`'s public
  `index.js`. This is mechanical (no behavior change), not a design
  decision, so it wasn't gated behind the live-discussion process — added
  directly, confirmed with the user first since it was an optional add-on
  beyond the doc-only scope.
