# Session 28 — 2026-07-23

## Goal

First implementation session since `docs/design/editor.md` was designed
(session 26) and hardened (session 27): roadmap item 1 of BACKLOG.md's
"packages/editor design roadmap" — the `core`-only mechanisms bundle
(`registerRule`'s `components` filter, generator `paramsDefaults`, the
dev-mode reads/writes `ctx` wrapper, `getComponentsForEntity`), plus the
pulled-forward `mods.js` → Plugin rename. A full plan for this exact scope
had been drafted in session 27 but shelved before approval when the user
redirected to revisit `editor.md` first; this session re-derived it against
the current code (read directly, not assumed from the design doc alone)
and ran it as five `/checkpoint-plan` checkpoints, each committed and
verified on its own before the next began.

## Key decisions

- **Components filter shape confirmed against `editor.md`'s exact spec**:
  `{ all?, any?, none? }`, each entry either a bare component-type string
  (presence-only) or `{ component, ...operators }` (presence + per-field
  partial-match comparisons via `equals`/`notEquals`/`gt`/`gte`/`lt`/`lte`/
  `in`/`notIn`) — not whole-object equality, so unrelated extra fields on
  the component data never break a match.
- **`registerEntityType`'s rewiring simplifies more than expected**: the
  old hand-rolled guard closure (`if (!hasComponent...) return; if
  (getComponent...type !== id) return;`) collapses to passing `handler`
  straight through to `registerRule` with a one-line `components` filter —
  no wrapper closure needed at all, confirming `editor.md`'s framing that
  this was sugar over the new primitive rather than a separate mechanism.
- **Dev-mode reads/writes wrapping is per-rule-entry, not per-dispatch**:
  since `reads`/`writes` are declared per rule, the `ctx` passed to
  `ruleFn` has to be rewrapped (or left alone) for each pipeline entry
  individually — `wrapCtxForRule(ctx, entry)` is called inside the
  dispatch loop, not once outside it. Priority-function evaluation in
  `dispatchExclusive` deliberately still uses the plain (unwrapped) `ctx`,
  not the rule's wrapped one — `editor.md` only specified wrapping around
  `ruleFn` itself, and forcing priority functions to also declare
  reads/writes wasn't part of that design.
- **`paramsDefaults` merge must preserve reference identity when no
  defaults are registered** — caught by an existing test
  (`mapgen.test.js`'s reference-equality check on `ctx.params`) failing
  after the first pass, which always spread into a new object regardless.
  Fixed to `entry.paramsDefaults ? { ...entry.paramsDefaults, ...params } :
  params`, so a generator with no `paramsDefaults` behaves byte-for-byte
  as before.
- **`cellularAutomataGenerator`'s buried defaults live in
  `zoneComposition.js`'s `carveCellularAutomata`, not in
  `cellularAutomataGenerator.js` itself** — confirmed by reading the
  generator file directly (it has no defaults of its own, just passes
  `ctx.params` straight through), so `DEFAULT_FILL_PROBABILITY`/
  `DEFAULT_PASSES`/`DEFAULT_WALL_THRESHOLD` are exported from
  `zoneComposition.js`.
- **`scripting-api.md`'s Plugin/Mod rename applied section-by-section, not
  as a blanket find-replace** — `editor.md`'s split (Plugin: dev-time,
  author-managed, statically-registered; Mod: still-undesigned,
  player-facing, runtime-toggleable) meant the module-format/dependency-
  version/save-slice sections renamed to Plugin, while the sandboxing and
  "Scope boundary: first-party vs. end-user mod distribution" sections'
  genuine end-user-mod language was left exactly as-is, since that's the
  still-unbuilt distribution concept, not this rename.
- **`packages/cli`'s half of the rename turned out to be a documentation-
  only change**: `packages/cli` has no scaffold code on disk yet (just a
  placeholder README), so the "`src/mods/` → `src/plugins/`" item BACKLOG.md
  named for it only existed as one line of prose in
  `docs/design/build-pipeline.md` — updated there, with the folder-per-
  plugin convention (`src/plugins/<pluginId>/`) noted alongside it, no
  actual code touched.
