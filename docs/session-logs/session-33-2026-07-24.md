## Key decisions

- **Numeric fields render as plain `<input type="number">`, no min/max/
  slider metadata layer.** The one real fork this session: `editor.md`'s
  audio-mixing prose calls the three volume fields "sliders," but
  `paramsDefaults`'s own design explicitly rejects a typed schema in favor
  of pure `typeof` inference. Raised with the user directly (not
  self-resolved) since it affects the primitive's API contract; chosen
  option keeps the primitive strictly to the documented rule and defers any
  slider polish to whichever future session (config UI, item 9) builds the
  real audio tab.
- **`LivePreview.svelte` needed zero new pure logic.** `paintLayer`
  (`renderLayers.js`) already takes a generic `{ col, row, text, color,
  background? }` command array decoupled from any zone/engine state — a
  1x1 swatch, a single assembled tile, and a small terrain grid all fit the
  same call shape with no new core code. The editor-side piece really is
  just canvas mounting + an `$effect` redraw, confirming `editor.md`'s "thin
  wrapper" framing literally rather than needing new abstraction.
- **No real downstream tool exists yet** (map editor etc. are items 5-9),
  so this session's own verification is dev-fixture demo scaffolding in
  `App.svelte` — three `LivePreview` instances and two `NarrowForm`
  instances, explicitly commented as throwaway checkpoint-4 scaffolding
  expected to be superseded once the real tools land, same pattern session
  32 used for `dev/sandbox/`.
- **The BSP-params demo form derives its defaults live from `api.registry`**
  (`get(api.registry, 'bsp').paramsDefaults`) rather than hand-copying the
  literal `{ minPartitionSize: 6, roomMargin: 1 }` — same "derive, don't
  hand-maintain" posture as `CORE_PLUGINS`/the mod manifest/the
  `components` filter elsewhere. The audio-mix demo's defaults are a plain
  literal, since `audioSettings.js` genuinely has no registered/derivable
  source — a game author would write that object directly.

## Work completed

Two checkpoints, each verified (tests + manual browser check) and committed
before starting the next:

1. **Live-preview rendering primitive.** `packages/editor/src/
   LivePreview.svelte` — commit `5a38ac0`. Props `{ commands, cols, rows,
   metrics, fontFamily, palette }`; sizes its canvas from `cellSize(metrics)`
   and redraws in full via `paintLayer` inside an `$effect` on any prop
   change (no dirty-tracking — cheap at preview scale, not a live game
   loop). Demo panel: a solid-color swatch, an assembled `@` tile via
   `resolveSymbol`, and a hardcoded 6x4 terrain grid standing in for a
   scratch zone/calibration grid.
2. **Narrow form primitive.** `packages/editor/src/narrowForm.js` (pure
   `inferFieldType`/`buildFieldSpecs`) + `NarrowForm.svelte` (controlled,
   callback-up via `onChange(key, value)`, same pattern as
   `PluginList`/`PluginServices`) — commit `48e8b4a`. Scoped to exactly the
   two documented consumers (generator `paramsDefaults`, audio mixing);
   keybindings/palette/tileset-calibration stay out per `editor.md`'s own
   rejection of those shapes. Demo panel: BSP params (live-derived) and
   audio mix (`{ master: 1, music: 0.7, sfx: 0.7 }`), both editable with
   isolated per-field state confirmed via manual browser interaction.

`packages/editor` test count: 46 → 50 (4 new: type inference, field-spec
