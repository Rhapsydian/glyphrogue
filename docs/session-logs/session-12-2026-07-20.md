# Session 12 — 2026-07-20

## Goal

Same conversation, continuing directly from session 11: work through item
G (AI/behavior-tree design) — how non-player actors decide what to do on
their turn.

## Key decisions

- **No new registration mechanism.** `act()` dispatches a core-shipped
  action (originally named `DecideAction`, renamed `TakeTurn` in session
  13) for any non-player actor; ordinary rules react to it, matched by
  component exactly the way `ExplodesOnDeath` is matched today. Core ships
  a handful of these as first-party default rules (`Wanders`,
  `ChasesPlayer`, `Flees`, `Guards`) — same relationship core has to its
  four built-in map generators, not a closed set.
- **Shared `ctx.findPath(from, to, opts)` pathfinding primitive** — AI
  rules call into it rather than each reimplementing A*/Dijkstra-map
  pathfinding, mirroring `rendering.md`'s shared shadowcasting precedent
  (which already covers AI perception, so this session only needed to add
  the pathfinding half).
- **Scope boundary**: this mechanism is for map-level, core-visible actors
  only. A custom battle screen's own private opponent logic (per session
  10's opacity model) is the screen author's concern, not this mechanism's.
