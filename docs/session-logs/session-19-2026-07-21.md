# Session 19 — 2026-07-21

## Goal

Roadmap item 19: the four built-in map-generation algorithms (BSP, cellular
automata, WFC, layered biome) registered through session 18's
`registerGenerator`/`GenerationContext` interface and composition
primitives, plus the optional world/region tier if scoped in.

## Key decisions

Three design questions the source doc left open were resolved live with the
user before planning, via `AskUserQuestion`:

**World/region tier: deferred, not built this session.** `getNeighborZone`
stays a caller-injected opaque lookup; no grid/coordinate system was built.
Revisit when a concrete game/session actually needs it.

**WFC: minimal single-cell version**, not a fuller sample-based WFC —
author-declared tile set + directed per-direction adjacency rules, lowest-
entropy collapse + forward-checking propagation, restart-on-contradiction.
A fuller sample-based WFC (pattern extraction, frequency weighting, real
backtracking) is backlogged as a future item (see `BACKLOG.md`).

**Layered biome: nearest-seed-point (Voronoi-like) partition** for the base
biome-region layer, not stacked/overlapping CA passes.

Two more design questions came from the user reviewing the plan itself,
before any code was written, and reshaped its architecture:

**Traversability became a shared composition primitive
(`ensureTraversable`), not per-generator logic.** The original plan only
fixed CA's well-known "disconnected floor pocket" problem, via bespoke
pruning logic local to that one generator, and left WFC/layered-biome with
a documented-but-unfixed limitation. The user pointed out this was the same
problem three times over, not three different problems. `ensureTraversable`
was pulled out as its own primitive in `zoneComposition.js`, alongside
`runConnectivityPass`, with two modes: `prune` (convert unreached, non-stamp
walkable cells to `'wall'` — right for CA's cave vocabulary, which has
nothing to lose by pruning) and `connect` (carve a corridor into a
disconnected pocket instead of destroying it — right for WFC and layered
biome, where the generated content itself, not just cave shape, is the
point). Kept distinct from `runConnectivityPass` since that primitive's
contract is specifically about a bounded, known `stamps` list, not a sweep
over the whole grid for leftover disconnected regions.

**Every algorithm became a region-scoped composable primitive with a thin
whole-zone generator wrapper on top**, matching the split
`carveCellularAutomata` (primitive, session 18) / the CA generator (thin
wrapper) already had. The user asked whether an author could compose a BSP
room-and-corridor dungeon that opens into a CA cave within one zone — the
answer was "not with the plan as drafted," since BSP/WFC/layered-biome were
each shaped as monolithic whole-zone generators. `carveBsp`, `collapseWfc`,
and `partitionBiomes` were reshaped to accept a `region` option and return
enough (room centers / entry point) for calling code to `connectCorridor`
between adjacent regions itself — composition is something an author's own
generator function does by calling multiple primitives in sequence, not a
meta-system core provides, matching `mapgen-and-editor.md`'s own framing.

## Work completed

- Checkpoint 1: `ensureTraversable(zone, { entryPoints, stamps, mode })` in
  `packages/core/src/zoneComposition.js` — `prune`/`connect` modes, both
  excluding all stamp bounds from handling (so `mayBeIsolated` content
  survives either mode). Also exported previously-private `isWalkableCell`/
  `nearestOpenCell` for reuse across the new generators. 5 new tests.
  Committed as `e30ebdf`.
- Checkpoint 2: `packages/core/src/bsp.js` — `carveBsp` (region-scoped
  recursive split/carve, connecting sibling rooms on the way back up the
  recursion — self-connecting by construction, no dependency on
  `ensureTraversable`) and `bspGenerator` (thin whole-zone wrapper). 8 new
  tests, including one proving the region-scoping composability case.
  Committed as `1277685`.
- Checkpoint 3: `packages/core/src/cellularAutomataGenerator.js` — thin
  wrapper over session 18's `carveCellularAutomata`, using
  `ensureTraversable`'s `prune` mode by default (`pruneUnreachable: false`
  opts out) to clean up the disconnected floor pockets raw CA carving is
  known for. 6 new tests. Committed as `c91d582`.
- Checkpoint 4: `packages/core/src/waveFunctionCollapse.js` — `collapseWfc`
  (region-scoped, lowest-entropy collapse + forward-checking propagation;
  a tile/direction pair with no declared adjacency rule is incompatible by
  default, not implicitly allowed — explicit, matching the "minimal WFC"
  decision; contradictions retry the whole attempt against the same seeded
  rng stream, never reseeding) and `wfcGenerator` (uses `ensureTraversable`'s
  `connect` mode, since a validly-collapsed region is exactly the content
  the algorithm exists to produce). 8 new tests, including a strict
  2-tile-checkerboard correctness check and a 3-tile "water/lava never
  touch" invariant. Committed as `be5ebe9`.
- Checkpoint 5: `packages/core/src/layeredBiome.js` — `partitionBiomes`
  (nearest-seed-point/Voronoi-like base layer, region-scoped) and
  `layeredBiomeGenerator` (stamps as the feature layer, then
  `ensureTraversable`'s `connect` mode — a biome region, like a WFC region,
  is deliberate content worth preserving rather than pruning). 8 new tests.
  Updated `docs/data-model.md`'s "Registration & API surface" table with
  all four generators, `ensureTraversable`, and the uniform `params.stamps`
  mechanism. Committed as `1c6ab12`.
- Full suite: 125/125 `node --test` cases passing (up from 91 at session
  start).

## Deferred / parked

- The optional world/region tier — deliberately deferred this session (see
  "Key decisions" above); revisit when a concrete need arises.
- A fuller sample-based WFC (overlapping NxN pattern extraction from a
  sample, frequency-weighted collapse, real backtracking search) — the
  minimal single-cell version built this session is enough for the design
  doc's stated use case, but is a real gap from "classic" WFC. New
  `BACKLOG.md` deferred item.
- No worked example/helper for composing multiple algorithms into one
  zone (e.g. an actual "BSP dungeon opening into a CA cave" sample
  generator) — the *capability* was built (every algorithm is region-scoped
  and returns composition-ready entry points), but nothing exercises it
  end-to-end yet. Would make a good, cheap addition whenever a concrete game
  or the editor first wants such a layout.

## Problem-solving notes

- The "disconnected biome patch that `ensureTraversable` connects rather
  than destroys" test needed a seed that genuinely produces an isolated
  floor pocket under a 2-biome (floor/wall) partition — not just an
  incidentally-already-connected one (same pitfall session 18's own notes
  flagged for its anchor-preference test). Rather than guess, a throwaway
  script (`packages/core/findSeed.mjs`, deleted after use, never committed)
  swept seeds 1–200 through `partitionBiomes` + a `prune`-mode
  `ensureTraversable` check to find one that actually changed cells when
  pruned, confirming a real disconnected pocket existed. Seed 2 was used in
  the final test.
